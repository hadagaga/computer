SYN Flood 向服务端发送大量连接请求，却不正常完成后续握手，使服务端维持大量未完成连接状态。理解它需要同时理解 TCP 三次握手、连接队列、超时和操作系统的资源管理。

## 核心问题

1. 服务端收到 SYN 后需要保存哪些状态？
2. 攻击为什么能够消耗连接队列和系统资源？
3. SYN Cookie、限速和队列调优分别解决什么问题？
